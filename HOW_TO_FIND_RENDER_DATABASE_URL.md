## üìç Where to Find It

### Method 1: Render Dashboard (Easiest)

1. **Go to Render Dashboard:**
   - Navigate to https://dashboard.render.com
   - Login to your account

2. **Find Your Database:**
   - Click on "Databases" in the left sidebar
   - Find your database: `myshp-db`
   - Click on it

3. **View Connection String:**
   - Scroll down to "Connections" section
   - You'll see:
     - **Internal Database URL** (for services on Render)
     - **External Connection String** (for local development)

4. **Copy the Internal Database URL:**
   - Format: `postgresql://username:password@hostname:port/database_name`
   - This is what Render automatically sets as `DATABASE_URL`

### Method 2: From Your Web Service

1. **Go to Your Backend Service:**
   - Click on `myshp-backend` service
   - Go to "Environment" tab

2. **Find DATABASE_URL:**
   - Look for `DATABASE_URL` in the environment variables list
   - This is automatically set when you link the database
   - Click "Reveal" to see the full connection string

## üîê Database URL Format

Your Render database URL will look like this:

```
postgresql://myshp_user:password@dpg-xxxxx-a.oregon-postgres.render.com:5432/myshp
```

**Components:**
- **Protocol:** `postgresql://`
- **Username:** `myshp_user` (from render.yaml)
- **Password:** Auto-generated by Render
- **Host:** `dpg-xxxxx-a.oregon-postgres.render.com` (Render's internal hostname)
- **Port:** `5432` (PostgreSQL default)
- **Database:** `myshp` (from render.yaml)

## ‚ö†Ô∏è Important Notes

### Internal vs External URL

**Internal Database URL (What Render Uses):**
- Used by services **on Render** (your backend service)
- Automatically set as `DATABASE_URL` environment variable
- Only accessible from Render's internal network
- Format: `postgresql://user:pass@internal-hostname:5432/dbname`

**External Connection String (For Local Development):**
- Used for connecting **from your local machine**
- Different hostname (external-facing)
- Requires SSL connection
- Format: `postgresql://user:pass@external-hostname:5432/dbname?sslmode=require`

### Auto-Set by Render

**You DON'T need to manually set `DATABASE_URL`!**

When you link your database to your web service in Render:
1. Render automatically creates the `DATABASE_URL` environment variable
2. It uses the **internal database URL** (for services on Render)
3. Your Django app automatically uses it

**How it works:**
- In `render.yaml`, you have:
  ```yaml
  - key: DATABASE_URL
    fromDatabase:
      name: myshp-db
      property: connectionString
  ```
- This tells Render: "Get the connection string from `myshp-db` database"
- Render automatically sets it when you deploy
